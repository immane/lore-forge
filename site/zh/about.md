# 关于 Lore Forge

## 什么是 Lore Forge？

Lore Forge 是一个**故事圣经引擎**——一个 AI 辅助的叙事设计框架，帮助创作者构建并维护鲜活、持续生长的故事世界。

它将故事创作视为**知识工程**：世界观构建、角色心理、时间线逻辑、记忆结构和叙事规划被组织成一个持久、可审查的系统，称为**故事圣经**（Story Bible）。

## 什么是故事圣经？

故事圣经是整个故事宇宙的唯一事实来源。它包含：

| 模块 | 内容说明 |
|--------|----------|
| **故事** | 愿景、主题、情感核心、时间线、章节大纲、结局设计 |
| **角色** | 心理档案、核心创伤、欲望、恐惧、声音签名、成长弧 |
| **地点** | 感官描述、氛围、功能、秘密、章节出场 |
| **知识** | 世界规则、术语表、正典事实、象征体系、情感地图 |
| **记忆** | 角色记忆图谱——每个角色记得什么（以及误记了什么） |
| **对话** | 带注释的对话，包含情感状态、潜台词和关系张力 |
| **任务 / 事件** | 任务依赖关系、事件链、选择与后果设计 |

你在本站阅读的所有内容都源自已确认的故事圣经。没有任何内容是临时编造的——每个细节都在知识层中有可追溯的来源。

## 如何使用本站

### 浏览项目

前往[项目列表](/zh/projects)查看所有可用的故事世界。每张卡片显示项目名称及其内容摘要。

### 阅读项目

点击任意项目即可探索其完整的故事圣经：

1. 从**概述**（项目摘要）开始
2. 浏览**故事**部分，了解主题、愿景和时间线
3. 认识**角色**——阅读他们的心理档案和成长弧
4. 查看**地点**，感受氛围和世界构建的细节
5. 深入**知识**部分——术语表、世界规则和象征体系
6. 阅读**导出**内容——小说章节和剧本草稿

### 阅读导出小说

许多项目在**小说**部分包含导出的叙事散文。这是从故事圣经生成的逐章草稿，提供传统的阅读体验。

## 设计理念

Lore Forge 建立在一个简单的理念之上：**好故事值得持久的地基。**

传统的 AI 故事生成将叙事视为一次性输出：输入提示，输出故事。但真正的故事——尤其是长篇游戏、视觉小说和互动小说——需要的不仅仅是一稿。它们需要一个能够：

- **在数百个场景中保持一致性**
- **追踪角色成长**，避免声音漂移
- **保持世界规则稳固**，无矛盾
- **支持迭代**——作者应该能够修订内容而不破坏正典

故事圣经方法使叙事知识变得持久、可审查、可复用、可安全更新。无论你是在阅读单个小说章节还是探索整个 RPG 世界，一切都可追溯到事实来源。

## 支持的类型

Lore Forge 适用于：

- **叙事型 RPG**——带有任务、选择和后果的故事驱动游戏
- **视觉小说**——基于路线的叙事，有角色分支和多重结局
- **互动小说**——基于选择树和解析器的互动故事
- **桌面角色扮演**——D&D 及其他 TTRPG 战役规划
