# Generate Dialogue

## 作用

`Generate Dialogue` 负责把场景定义转化为带潜台词、情绪状态和关系张力的对白脚本。

## 触发条件

- 场景定义已存在
- 用户要求为某个场景写对白
- Narrative Designer 将场景移交给 Dialogue Writer
- 用户显式要求进入 Phase 6

## 负责 Agent

Dialogue Writer

## 依赖

- 场景定义文件
- 相关角色档案
- `knowledge/glossary.md`
- `knowledge/rules.md`
- `knowledge/symbolism.md`
- 既有对白文件

## 工作流结构

### Step 1：分析场景

1. 谁在说话？
2. 他们当前状态是什么？
3. 关系图谱和权力结构是什么？
4. 这段对白要完成什么情绪变化？

### Step 2：构建对白

5. 每句台词的意图是什么？
6. 字面意思和潜台词是什么？
7. 这句话是否改变了主导权？
8. 这真的是这个角色会说出来的话吗？

### Step 3：自审

9. 去掉说话人标签后还能分辨角色吗？
10. 术语使用是否一致？
11. 角色有没有说出不该知道的内容？
12. 潜台词密度是否足够？
13. 是否还能更短更有力？

### Step 4：标注

14. 给每句台词加上说话人、情绪状态、潜台词和关系张力说明

## Inputs

- 场景定义
- 角色档案
- 术语表与规则表

## Outputs

- `dialogue/{chapter}-{scene}.md`
- 角色关系更新
- `dialogue/_index.md` 更新

## 推荐输出格式

```markdown
**角色 A** [状态：...] 
{潜台词：...}
台词内容

**角色 B** [状态：...]
{潜台词：...}
台词内容
```

## 校验规则

- 每个说话角色都必须有目标
- 角色不能引用自己未经历过的事件
- 两个角色的声音不能完全相同
- 世界内术语必须与 glossary 一致
- 匿名对白不可接受
- 如果台词违背角色心理档案，应中止并回交 Character Therapist
